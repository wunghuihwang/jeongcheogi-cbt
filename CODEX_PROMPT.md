==================================================
첨부 기출문제 파일 처리 — 최우선 요구사항
==================================================

사용자가 작업공간에 첨부한 정보처리기사 필기 기출문제 파일 전체를 이 프로젝트의 원본 문제은행으로 사용해라.

현재 제공된 자료:

- 2022년 정보처리기사 필기 1회
- 2022년 정보처리기사 필기 2회
- 2022년 정보처리기사 필기 3회
- 2023년 정보처리기사 필기 1회
- 2023년 정보처리기사 필기 2회
- 2023년 정보처리기사 필기 3회
- 2024년 정보처리기사 필기 1회
- 2024년 정보처리기사 필기 2회
- 2024년 정보처리기사 필기 3회
- 2025년 정보처리기사 필기 1회
- 2025년 정보처리기사 필기 2회
- 2025년 정보처리기사 필기 3회

총 4개년, 12회분이다.

작업공간의 upload 디렉터리와 하위 디렉터리를 재귀적으로 탐색하여 다음 파일을 모두 찾아라.

- ZIP
- PDF
- JSON
- CSV
- 이미지

파일명이 한글 자모 분리 형식으로 저장되어 있을 수 있으므로 파일명을 직접 하드코딩하지 말고 확장자와 실제 파일 구조를 기준으로 탐색해라.

ZIP 파일은 안전한 임시 디렉터리에 압축 해제한다.

ZIP Slip 공격을 막기 위해 다음 경로를 거부한다.

- 절대경로
- `../`가 포함된 경로
- 압축 해제 대상 디렉터리 밖으로 벗어나는 경로

원본 첨부 파일을 삭제하거나 수정하지 마라.

==================================================
중복 파일 처리
==================================================

2025년 1·2·3회 PDF는 ZIP 내부에도 있고 별도 PDF로도 제공되어 있으므로 중복될 가능성이 높다.

모든 원본 파일에 SHA-256 해시를 생성해라.

다음 기준으로 중복을 판정한다.

1. 파일 SHA-256이 동일한 경우
2. 연도·회차가 동일하고 추출된 문제 내용이 동일한 경우
3. 정규화한 문제 본문의 해시가 동일한 경우

중복 파일은 데이터베이스에 두 번 넣지 마라.

다만 원본 파일 이력은 `source_documents` 테이블에 남기고 다음 관계를 기록해라.

- 원본 파일명
- SHA-256
- 중복 여부
- 중복 대상 문서 ID
- 업로드 시각
- 연도
- 회차
- 페이지 수
- 추출 상태

==================================================
원본 PDF 보관
==================================================

원본 PDF는 Supabase Database에 바이너리로 직접 넣지 말고 Supabase Storage에 저장해라.

비공개 Storage Bucket:

exam-source-pdfs

저장 경로:

exam-source-pdfs/{year}/{round}/{normalized_filename}.pdf

Bucket은 공개하지 마라.

인증된 사용자도 원본 PDF를 무조건 직접 다운로드할 수 없게 하고, 필요한 경우 서버에서 권한을 확인한 후 짧은 만료시간의 Signed URL을 생성해라.

원본 파일 메타데이터는 `source_documents` 테이블에 저장한다.

필드:

- id
- original_filename
- normalized_filename
- storage_path
- mime_type
- file_size
- sha256
- exam_year
- exam_round
- page_count
- extraction_method
- extraction_status
- question_count
- is_duplicate
- duplicate_of
- error_message
- imported_at
- created_at

==================================================
PDF 문제 추출
==================================================

첨부 PDF의 내용을 읽어 모든 문제를 구조화된 데이터로 변환해라.

원칙:

- 문제 본문을 임의로 요약하거나 바꾸지 않는다.
- 선택지 내용을 임의로 바꾸지 않는다.
- 코드의 들여쓰기와 줄바꿈을 보존한다.
- SQL의 줄바꿈과 키워드를 보존한다.
- 표는 구조화된 JSON 또는 Markdown으로 보존한다.
- 문제에 포함된 이미지와 다이어그램을 누락하지 않는다.
- 원본 페이지 번호를 기록한다.
- 정답이 PDF에 있으면 정확히 연결한다.
- 정답을 찾을 수 없으면 임의로 추측하지 않는다.
- 추출 실패 문제는 별도로 표시하고 관리자 검토 대상으로 남긴다.

처리 순서:

1. PDF에 텍스트 레이어가 있는지 확인
2. 텍스트 레이어가 있으면 레이아웃을 보존하여 추출
3. 텍스트 추출 품질이 낮은 페이지만 OCR 수행
4. 표·코드·수식·이미지 영역 별도 처리
5. 문제 번호를 기준으로 문제 분리
6. A·B·C·D 또는 ①·②·③·④ 선택지 분리
7. 정답표가 있으면 문제 번호와 정답 연결
8. 연도·회차·과목·문제 번호 연결
9. 추출 결과 검증
10. 검증을 통과한 문제만 Supabase에 적재

사용 가능한 도구를 확인하여 다음 계열을 활용할 수 있다.

- pdftotext
- pdfplumber
- pypdf
- PyMuPDF
- OCR
- 이미지 렌더링
- OpenCV

OCR은 전체 PDF에 무조건 사용하지 말고 텍스트 추출이 불가능하거나 품질이 낮은 페이지에만 사용한다.

==================================================
문제 이미지 처리
==================================================

문제에 다음 자료가 포함되어 있으면 이미지 자산으로 보존한다.

- UML 다이어그램
- 네트워크 구성도
- 자료구조 그림
- 그래프
- 표
- 수식 이미지
- 코드가 이미지로 삽입된 경우
- ERD
- 페이지 교체 과정 그림

이미지는 다음 비공개 또는 읽기 전용 Storage Bucket에 저장한다.

question-assets

경로:

question-assets/{year}/{round}/{question_no}/{asset_name}

`question_assets` 테이블:

- id
- question_id
- source_document_id
- page_number
- asset_type
- storage_path
- alt_text
- display_order
- width
- height
- sha256
- created_at

이미지만으로 된 문제는 이미지 한 장만 표시하고 끝내지 말고, 가능한 경우 OCR 텍스트도 함께 저장한다. 그러나 OCR 결과로 원본 이미지를 대체하지 않는다.

==================================================
과목 및 문제 번호 매핑
==================================================

한 회차가 100문제로 구성된 경우 다음 기준으로 과목을 매핑한다.

- 1~20번: 소프트웨어설계
- 21~40번: 소프트웨어개발
- 41~60번: 데이터베이스구축
- 61~80번: 프로그래밍언어활용
- 81~100번: 정보시스템구축관리

단, PDF에 과목 정보가 명시되어 있으면 원본 정보를 우선한다.

회차별 문제 수가 100개가 아니거나 번호가 누락되면 자동으로 억지 보정하지 마라. 해당 회차를 `needs_review` 상태로 만들고 누락 번호를 보고해라.

- 문제 의미 변경
- 선택지 표현 임의 수정
- 문제 난이도 조정
- 정답을 유도하는 강조 추가
- 근거 없이 오탈자 수정
- 정답 임의 생성

원본과 정규화된 값을 모두 저장하여 언제든 비교할 수 있게 한다.

==================================================
Supabase 적재 API
==================================================

모든 문제를 Supabase에 저장하는 관리자 전용 Import API를 구현해라.

Route:

POST /api/admin/import-exams

기능:

1. 첨부 파일 검색
2. ZIP 안전 압축 해제
3. PDF SHA-256 계산
4. 중복 검사
5. PDF Storage 업로드
6. 텍스트 추출
7. 문제·선택지·정답 분리
8. 이미지 자산 추출
9. Zod 검증
10. 데이터베이스 Transaction 또는 안전한 Upsert
11. 적재 결과 반환

권한:

- 로그인한 관리자만 호출 가능
- service role 키는 서버에서만 사용
- 일반 사용자는 Import API 호출 불가
- 관리자 여부를 클라이언트 입력만으로 판단하지 않음

응답 예시:

{
"documentsFound": 15,
"uniqueDocuments": 12,
"duplicateDocuments": 3,
"examSetsImported": 12,
"questionsImported": 1200,
"questionsNeedingReview": 0,
"missingQuestions": [],
"errors": []
}

위 수치는 예시다.

실제 결과는 반드시 추출 및 검증 결과를 바탕으로 계산하고, 1,200문제라고 미리 고정하지 마라.

대용량 작업이 서버리스 요청 제한시간을 초과할 수 있으므로 다음 두 방식도 함께 제공한다.

1. 로컬 CLI Import 스크립트
2. 관리자 API

권장 CLI:

pnpm import:exams

CLI는 환경변수로 Supabase에 연결해 ZIP·PDF 전체를 처리한다.

Import API는 이미 구조화된 JSON을 재적재하거나 소량의 파일을 추가할 때 사용한다.

중간 구조화 파일
==================================================

PDF에서 바로 DB로 넣지 말고 검증 가능한 중간 JSON을 생성해라.

경로:

data/imported/{year}/{round}/questions.json

구조 예시:

{
"year": 2022,
"round": 1,
"sourceDocument": "...",
"questions": [
{
"questionNo": 1,
"subject": "소프트웨어설계",
"questionText": "...",
"choices": {
"A": "...",
"B": "...",
"C": "...",
"D": "..."
},
"correctAnswer": "B",
"sourcePage": 1,
"assets": [],
"needsReview": false
}
]
}

중간 JSON도 Zod로 검증한다.

Import를 다시 실행해도 동일한 문제가 중복 삽입되지 않도록 멱등성을 보장한다.

==================================================
검증 스크립트
==================================================

다음 명령을 구현해라.

pnpm extract:exams
pnpm validate:exams
pnpm import:exams
pnpm report:exams

`extract:exams`

- ZIP과 PDF 탐색
- PDF 추출
- 구조화 JSON 생성
- 이미지 자산 생성

`validate:exams`

- 연도별 회차 확인
- 회차별 문제 수 확인
- 문제 번호 1~100 확인
- 선택지 4개 확인
- 정답 유효성 확인
- 중복 확인
- 빈 문제 확인
- OCR 품질 확인

`import:exams`

- Supabase Storage 업로드
- Supabase Database Upsert
- 재실행 시 중복 방지

`report:exams`

다음 표를 출력한다.

| 연도 | 회차 | 문제 수 | 정답 수 | 이미지 문제 | 검토 필요 | 상태 |

누락 문제 번호와 오류 파일도 함께 출력한다.

==================================================
앱의 출제 방식
==================================================

앱은 첨부된 2022~2025년 문제은행을 사용하여 출제한다.

지원 모드:

1. 연도·회차 그대로 풀기
2. 특정 과목만 풀기
3. 5·10·20·50문제 랜덤 풀이
4. 100문제 실전 CBT
5. 오답만 다시 풀기
6. 미풀이 문제만 풀기
7. 즐겨찾기 문제만 풀기
8. 코드 문제만 풀기
9. SQL 문제만 풀기
10. 여러 연도를 섞은 랜덤 모의고사

연도·회차 모드에서는 원본 문제 순서를 유지한다.

랜덤 CBT에서는 다음 조건을 지킨다.

- 총 100문제
- 과목별 20문제
- 동일 문제가 한 시험에 중복되지 않음
- 가능하면 연도·회차를 균형 있게 섞음
- 사용자가 이미 여러 번 맞힌 문제보다 미풀이·오답 문제 우선 가능
- 답안 제출 전 정답 비공개

콘텐츠 출처 표시
==================================================

각 문제 화면과 결과 화면에 다음 정보를 표시한다.

- 2022년 1회
- 문제 번호
- 과목
- 원본 페이지
- 자료 출처

실제 첨부 PDF에서 추출한 문제는 다음처럼 저장한다.

source_type: uploaded_exam_pdf

자체 제작 문제와 섞지 마라.

필터에서 다음을 구분할 수 있게 한다.

- 첨부 기출문제
- 자체 제작 문제
- 기출 변형 문제

==================================================
해설 처리
==================================================

PDF에 해설이 포함되어 있으면 해설 원문을 함께 추출한다.

PDF에 정답만 있고 해설이 없다면 다음 원칙을 적용한다.

- 문제와 정답은 먼저 정상 적재
- explanation_status를 `missing`으로 기록
- 임의로 원본 해설이 존재하는 것처럼 표시하지 않음
- 앱에서는 “원본 자료에 해설이 없습니다”라고 표시 가능
- 추후 관리자가 상세 해설을 추가할 수 있게 구현
- 해설 추가용 JSON Import 기능 제공

서비스 실행 중 OpenAI API를 호출하지 않는다.

Codex가 작업 과정에서 상세 해설을 작성한다면 별도 필드에 다음처럼 명확히 표시한다.

explanation_source: codex_authored

원본 PDF 해설이면 다음처럼 표시한다.

explanation_source: source_pdf

정답과 해설에 오류가 없는지 검증 상태를 기록한다.

==================================================
첨부 출제기준 PDF 처리
==================================================

사용자가 별도로 제공한
‘Crownix HTML5 Viewer.pdf 적용기간 2026.1.1~2026.12.31’
‘Crownix HTML5 Viewer2.pdf 적용기간 2023.1.1~2025.12.31’
PDF를 시험 범위의 최우선 기준으로 사용한다.

인터넷에서 임의로 요약한 범위를 우선하지 마라.

출제기준 PDF 원본도 Supabase Storage의 비공개 Bucket에 저장한다.

Bucket:

exam-standards

다음 데이터를 구조화해 저장한다.

`exam_standards`

- id
- title
- effective_from
- effective_to
- source_document_id
- raw_text
- parsed_structure
- sha256
- created_at

`exam_standard_items`

- id
- exam_standard_id
- subject_name
- major_category
- middle_category
- minor_category
- details
- source_page
- display_order

출제기준 PDF 원문과 구조화된 범위를 모두 보존한다.

문제를 과목·개념·태그로 분류할 때 이 출제기준의 항목을 기준으로 매핑한다.

출제기준에 없는 내용이라고 판단되면 문제를 삭제하지 말고 다음처럼 표시한다.

scope_status: needs_review
